# 会社PC向けビュー改修方針

## 目的

会社支給のノートPCで見たときに、画面内で情報がきちんと読めるようにする。

現在の画面は、大きいモニターでは成立しているが、会社PCのブラウザ100%では余白や大きなカードが目立ちすぎて、肝心の情報面積が足りなくなる。

この改修では、表示をただ縮めるのではなく、**左に header + map、右に chat が独立して縦に貫く分割**へ寄せる。

## 現状理解

観測した会社PCのブラウザ幅は次の通り。

- `window.innerWidth`: `1258px`
- `document.documentElement.clientWidth`: `1243px`
- `window.devicePixelRatio`: `1.50`

この幅では、以下が問題になりやすい。

- 大きな見出しが情報を押しのける
- ヘッダーの3カードが場所を使いすぎる
- 地図ポップアップが大きく、地図を隠しやすい
- 右チャット欄の見やすさが足りない

## 改修の基本方針

### 1. 右チャット欄を主役として扱う

画面の分割方法を見直し、右のチャット欄が上から下まで独立して見える幅を確保する。

### 2. 上部の情報は薄くする

ヘッダーは主役にしない。  
最悪、かなり薄くしてよい。

### 3. 文字のジャンプ率を下げる

大見出しを強くしすぎず、本文も少し小さくする。  
画面全体を穏やかな情報密度に寄せる。

### 4. ポップアップを小さくする

ポップアップはアイコンなどの視覚的な手がかりを残しつつ、カード自体は小さくする。  
地図の露出面積を優先する。

### 5. 地図を広く見せる

地図は左カラムの下側に置き、かぶせる情報は最小限にする。  
地図の上に大きなカードを乗せすぎない。

## 変更の優先順位

1. ヘッダーの3カードをやめる
2. 見出しと本文の文字サイズを下げる
3. 左に header + map、右に chat が独立する分割に変える
4. ポップアップを小さくする
5. 地図の見え方を再調整する

## 判断基準

次が満たせれば、会社PC向けの改修として合格とする。

- 100%表示で見ても情報が読める
- 右チャット欄が使いやすい
- 地図が十分に見える
- ブラウザ縮小に頼らなくてよい

現時点では、会社PC実機でこの基準にかなり近いところまで到達している。

## 補足

- これは既存UIの微調整ではなく、会社PC向けの見せ方の再設計に近い
- 大画面向けの見せ方も、左に header + map、右に chat が独立する分割に寄せる
- `web/debug-width/` は測定用として残してよい
