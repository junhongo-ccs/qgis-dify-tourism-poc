# 会社PC向けビュー実装計画

## 目的

会社PC向けの見せ方を、既存UIに悪影響を与えずに実装する。

## 実装の考え方

### 方針

- 既存のメインUIは壊さない
- 会社PC向けの見え方を先に作る
- 必要なら別ルートまたは別ページで切り出す
- 大画面でも会社PCでも、左に header + map、右に chat が独立する分割を優先する

### 実装対象

- ヘッダー
- 地図エリア
- 右チャット欄
- ポップアップカード
- 文字サイズと余白

## 画面構成の方針

### 上部

- ヘッダーは左カラムの上部だけに置く
- 3カードは表示しない
- タイトルと最小限の説明だけ残す

### 中央

- 地図を主役にする
- 地図を隠す大きなカードを減らす
- ポップアップは小さくする

### 右側

- チャット欄を独立した縦列として扱う
- 会話ログが読める高さを確保する
- 入力欄を使いやすくする

## 文字設計

会社PC向けでは、次を行う。

- 見出しを小さくする
- 本文も小さくする
- ポップアップの文字も控えめにする
- 文字の強弱を抑える

## レイアウト分岐

まずは、次のどちらかで始める。

### 案A. 別ページ

会社PC用の別ページを作る。

良い点:

- 既存UIと切り離しやすい
- 調整が分かりやすい
- 消すときも簡単

### 案B. 条件分岐

同じアプリ内で画面幅を見て分岐する。

良い点:

- 導線が少ない
- 管理がまとまる

### 推奨

最初は別ページで試す。  
会社PC向けの見え方を固めてから、必要なら本体に統合する。

ただし、本体側の構造方針も同じく「左に header + map、右に chat」の2分割を基本にする。

## 検証方法

会社PCで次を確認する。

1. ブラウザ100%
2. ブラウザ90%
3. 地図を選択した状態
4. チャットが数件たまった状態
5. ポップアップが出た状態

## 完了条件

- 大見出しが邪魔にならない
- 右チャット欄が上から下まで独立して見える
- 地図が左下に安定して収まる
- 会社PCの100%表示で成立する

## 進め方

1. 改修方針を確定する
2. 会社PC向けの別ページか分岐を決める
3. 文字サイズと余白を先に詰める
4. 右チャット欄の分割を整える
5. ポップアップを小さくする
6. 会社PCで見直す
